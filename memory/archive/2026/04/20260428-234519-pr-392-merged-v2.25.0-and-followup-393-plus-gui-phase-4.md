## What landed in v2.25.0 (PR #392)
- `noetl.server.api.mcp` package with three endpoints under `/api`:
  - `POST /mcp/{path:path}/lifecycle/{verb}` — resolves `spec.lifecycle.{verb}` to an Agent path and dispatches via `/api/execute`.
  - `POST /mcp/{path:path}/discover` — agent strategy via `spec.discovery.refresh_via` OR direct fetch via `spec.discovery.tools_list_url` with diff + re-register.
  - `GET /catalog/{path:path}/ui_schema` — workload form inferred from playbook YAML.
- Workload form inference (`ui_schema.py`): plain PyYAML structure + regex scan over raw text for `# ui:secret` / `# ui:enum=[...]` / `# ui:credential=glob` / `# ui:description=...` directives. No ruamel dep.
- Module-level docstring documents the auth-as-playbook design (no Python `Depends(require_admin)`; authorization governed by the existing `api_integration/auth0/check_playbook_access` flow against `auth.playbook_permissions` because lifecycle verbs already resolve to Agent playbook paths).

## What's queued in PR #393 (follow-up)
Eight Copilot pass-2 findings on commit `bf636e18` of the merged PR:
- `service.fetch_mcp_resource` shallow-copies parsed payload before injecting `_catalog` (fixes CatalogService cache pollution).
- `service._fetch_entry` maps real exceptions to 503 instead of masking everything as 404. `HTTPException` from the catalog service propagates unchanged.
- `service.dispatch_lifecycle` rejects `workload_overrides` keys that would clobber reserved fields (`mcp_resource`, `verb`) with 422.
- `endpoint._host_is_safe_for_discovery` is async and uses `loop.getaddrinfo` instead of blocking `socket.getaddrinfo`.
- `schema.py` drops unused `field_validator` / `model_validator` imports; `UiSchemaField` docstring fixed to point at `ui_schema.infer_ui_schema`.
- `ui_schema._TOP_KEY_RE` accepts indents of 2+ spaces (was exactly 2). `_DIRECTIVE_INLINE_RE` uses non-anchored `finditer` with a lookahead so multiple `# ui:` directives on one line all get parsed.
- Four new unit tests covering the new behaviour: workload-overrides reserved-keys reject, workload-overrides clean merge, multiple directives on one line, 4-space indent recognition.

## What's queued in GUI PR (Phase 4)
