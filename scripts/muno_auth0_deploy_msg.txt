Round goal
----------
Two coupled deliverables in one round:

  (a) Wire Auth0 authentication into the muno React app (SPA mode,
      @auth0/auth0-react). Sidebar Sign in / Sign out, Bearer token on
      API calls, user_uid from Auth0 sub claim into chat events.
      Guest mode preserved as fallback. NO server-side JWT validation
      in this round — frontend-trusted user_uid is acceptable for v1;
      validation is a follow-up.

  (b) Deploy the resulting container to muno.mestumre.dev behind the
      project's existing GKE + Cloudflare ingress. Push image to
      ghcr.io/noetl/muno. Apply K8s manifests. Capture Ingress IP.
      Output Cloudflare DNS instructions for Kadyapam. Resume to live
      smoke after Kadyapam confirms DNS is added.

Pre-handoff (Kadyapam, before firing this round)
------------------------------------------------
~10 minutes total. Round AMBERs at phase 1 if missing.

  1. Register an Auth0 SPA app named 'Adiona / muno':
     - Allowed Callback URLs: https://muno.mestumre.dev/callback, http://localhost:5173/callback
     - Allowed Logout URLs: https://muno.mestumre.dev, http://localhost:5173
     - Allowed Web Origins (incl. CORS): same two URLs
     - Optional API audience: https://api.muno.mestumre.dev (for
       future server validation)
     - Capture Domain + Client ID.
  2. Provision GCP secrets:
     - muno-auth0-domain
     - muno-auth0-client-id
     - muno-auth0-audience
     - Grant noetl-worker-mcp `roles/secretmanager.secretAccessor` on each.
  3. Update google-maps-widget-key referrer restrictions to include
     `https://muno.mestumre.dev/*` (in addition to the existing
     mestumre.dev / gateway.mestumre.dev / *.pages.dev entries).

Mid-round handoff (DNS — Kadyapam, after phase 7)
-------------------------------------------------
Phase 7 outputs the Ingress external IP. Kadyapam adds the DNS record
at Cloudflare:

  - Cloudflare dashboard → `mestumre.dev` zone → DNS → Records → Add.
  - Type: A
  - Name: muno
  - IPv4: <Ingress IP from phase 7>
  - Proxy status: Proxied (orange cloud) — Cloudflare TLS + WAF edge.
  - TTL: Auto.
  - SSL/TLS mode for the zone: confirm Full (Strict) — should already be.

After DNS lands (1-2 min propagation), Kadyapam tells codex 'DNS added'
to resume phase 8 (live smoke).

Architectural rules
-------------------
  - Auth0 frontend-only. NO server-side JWT validation in this round.
  - Guest mode preserved as fallback.
  - Firestore security rules unchanged from v1 permissive.
  - Container image at ghcr.io/noetl/muno (mirror noetl/noetl).
  - Cloudflare proxy mode recommended (orange cloud). Origin cert via
    cert-manager Let's Encrypt; Cloudflare → origin uses Full (Strict).
  - Never commit Auth0 secrets or any token bytes.
  - muno PR + ops PR (if applicable) via standard flow.
  - ai-meta pointer bumps local-only.
  - Cloudflare DNS update is Kadyapam-side; codex outputs instructions.
