Tool surface (6 tools)
----------------------

  - set_doc — write/upsert at a path. Inputs: path, data, optional merge.
  - get_doc — read a doc. Inputs: path.
  - query_collection — runQuery with where/order_by/limit clauses.
  - delete_doc — delete (used for smoke teardown).
  - append_event — APPEND-ONLY insert into `<thread_path>/events` with
    transactional monotonic seq. MANDATORY header redaction on the
    event payload (Authorization, X-Goog-User-Project, x-figma-token,
    cookies, anything matching `(?i)(token|secret|password|api[-_]?key)`).
    No skip-redaction flag.
  - replay_events — read an event range, optionally filtered by type.
    No agent-diff in this round; that's Round 4 territory.

Generic, NOT trip-planner-specific
----------------------------------
The MCP tools CRUD whatever paths the caller hands them. The
trip-planner-specific `chat_threads/{threadId}/events` hierarchy is a
CONVENTION the Round 4 agent will follow. Don't bake the convention
into the playbook — keep the tools reusable for any future agent.

Implementation notes
--------------------
- Use Firestore REST API directly. Auth via
  `google.auth.default(scopes=['https://www.googleapis.com/auth/datastore'])`
  → SA token from metadata server (Workload Identity). Same pattern
  mcp/google-places uses.
- Firestore REST wraps values as `{stringValue: ...}`, `{integerValue: ...}`
  etc. Hand-roll a small encoder/decoder INSIDE the playbook — do NOT
  introduce `google-cloud-firestore` as a new Python dep on the worker
  image without separate approval.
- `append_event` uses a Firestore transaction (beginTransaction →
  query for max seq → assign max+1 → write → commit). This is what
  guarantees no two concurrent appends collide on seq.
- Header redaction is recursive over the event payload. Sensitive
  keys are replaced with the literal string `<REDACTED>`. Never
  recurse INTO redacted values. Smoke E in phase 3 proves this.

scripts/firestore_replay.sh
---------------------------
Operator-side helper. Runs locally, uses
`gcloud auth print-access-token` for the operator's context (NOT the
worker SA). Three modes:
  - events <thread_path> [--from N] [--to N] [--type-filter csv]
  - thread-list [--limit N]
  - doc <doc_path>
No agent-diff in this round; Round 4 extends with `replay --against-agent`.

Architectural rules
-------------------
  - WI auth only. No token secret.
  - Generic tools — no trip-planner data model baked in.
  - Mandatory header redaction in append_event. No bypass flag.
  - Transactional seq assignment. No eventual-consistency hacks.
  - All smoke writes scoped to `_smoke/...`. Cleaned at end of phase 3.
  - No new Python dep on worker image (hand-roll Firestore JSON
    encoding/decoding).
  - No release cut. No git push from ai-meta.
  - Never log full doc data beyond ~200 chars per assertion.
