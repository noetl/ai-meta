Tool surface (additions)
------------------------

  - create_order
      Inputs: offer_id, passengers[], optional payment_amount/payment_currency.
      Endpoint: POST /air/orders
      Payment: { type: "balance", amount: <offer.total_amount>, currency: <offer.total_currency> }
      Validation: reject empty passengers, reject expired offers
      (fetch offer first; check expires_at) before POSTing.

  - get_order
      Inputs: order_id.
      Endpoint: GET /air/orders/{id}
      Pass-through.

  - list_orders
      Inputs (optional): created_at_after, passenger_name, booking_reference.
      Endpoint: GET /air/orders (cursor pagination, first page only)
      Returns up to 50 results.
