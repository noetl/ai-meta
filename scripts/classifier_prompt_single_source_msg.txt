Context:
  After Phase 4 (Ollama provider), the classifier system_prompt is duplicated
  across THREE places in the travel runtime:
    1. classify_via_http_provider's Python `system_prompt` constant
    2. classify_via_vertex_mcp's payload.arguments.system inline string
    3. classify_via_ollama_mcp's payload.arguments.system inline string

  This was flagged as design debt in the hotels/activities round, then
  promoted to load-bearing in the Phase 4 close-out memo. Every schema
  extension since (cityCode, latitude, longitude) has touched all three.

  This round dedupes by lifting the prompt into a single workload field
  `workload.classifier_system_prompt` (multi-line YAML literal block scalar
  `|` style), referenced by all three branches via Jinja. Free side benefit:
  callers can override via workload for prompt A/B testing without forking.
