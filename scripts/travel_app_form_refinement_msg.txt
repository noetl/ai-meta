Context:
  Travel render_* widget trees are read-only. Once a user sees flights /
  hotels / locations / activities / friendly-error widgets, refining the
  search means typing a new prompt. This round adds an app:form refinement
  widget appended to each render_* tree. The form's fields are pre-populated
  from classify_intent values; the submit button emits a `key: "command"`
  widget event with a templated travel query that gets substituted at click
  time from the current field values. Both terminal and canvas already
  handle `key: "command"` events.

  The ONLY missing primitive is template substitution in AppForm.tsx —
  current click handler emits either button.event.value (static) or the
  values dict (object). Neither builds a string command from form values.

Patch design (Codex implements; full brief in task JSON design_brief_for_codex):

  repos/gui/src/components/widgets/AppForm.tsx
    Click handler change. If button.event.value is a string with `{fieldId}`
    placeholders, substitute each from values dict (empty string if missing).
    If button.event.value is non-string (object/array) or undefined, preserve
    existing behaviour.

    Pseudo:
      const rawValue = button.event?.value;
      let emittedValue: unknown = rawValue ?? values;
      if (typeof rawValue === 'string') {
        emittedValue = rawValue.replace(/\{(\w+)\}/g, (_match, fieldId) => {
          const fieldValue = values[fieldId];
          return fieldValue !== undefined && fieldValue !== '' ? String(fieldValue) : '';
        });
      }
      onWidgetEvent({ event: 'onFormSubmit', key: button.event?.key || 'onFormSubmit', value: emittedValue });

    Add unit test for the four shapes: static string, placeholder substitution,
    missing field, non-string fallback.

  repos/ops/automation/agents/travel/runtime.yaml
    AFTER the gui PR is merged AND deployed to GKE, append app:form widget
    to each render_* widget tree's app:column children list. Per-intent
    field/template shapes are in the design brief.

    render_flights:    origin/destination/departureDate/adults
    render_hotels:     cityCode/departureDate
    render_locations:  keyword
    render_activities: keyword
    render_amadeus_failure: introspect classify_intent.intent at render time
