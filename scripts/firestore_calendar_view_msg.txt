Calendar event document schema
------------------------------
Stored at `users/{uid}/trips/{tripId}/events/{eventId}` (or
`chat_threads/{threadId}/trip/events/{eventId}` for guest threads):

  {
    type: 'flight_depart' | 'flight_arrive' | 'check_in' | 'check_out' |
          'activity' | 'user_note',
    start_at: ISO 8601 with tz offset,
    end_at: ISO 8601 with tz offset, nullable,
    timezone: IANA name,
    title: short label,
    location: free string,
    notes: nullable string,
    source_order_id: nullable (Duffel order id),
    source_hotel_id: nullable,
    created_at, updated_at: server timestamps,
    google_calendar_event_id: null  // reserved-but-unused in v1
  }

Document in muno/docs/architecture/data-model.md.

New widget template — calendar_view
-----------------------------------
Schema: `muno/playbooks/widget-contract/calendar_view.schema.json`

Payload:
  {
    trip_id,
    events_path,                 // Firestore collection path
    display_events?,             // denormalized array (static mode)
    editable,                    // bool
    empty_state_text?
  }

Variants: `compact` | `full` | `read_only`.

Mutual exclusion: `display_events` (static snapshot) XOR live subscribe
via `events_path`. Agent picks based on context:
  - Inside `itinerary_summary` → static `display_events`.
  - Standalone "show me my schedule" → live `events_path`.

Run `npm run contracts` after adding the schema to regenerate
`muno/src/contracts/widgets.ts`.

Agent extension (muno/playbooks/itinerary-planner.yaml)
-------------------------------------------------------
- After `mcp/duffel.create_order` succeeds: write flight_depart +
  flight_arrive event docs for each segment. Path follows the rule
  above (user_uid or guest).
- After hotel pick confirmation: write check_in (default 15:00 local)
  + check_out (default 11:00 local) docs. location = full hotel address.
- After user-typed note that the LLM extracts as a calendar request
  ('add a 3pm meeting on Wednesday'): write user_note doc.
- When emitting `itinerary_summary`: ALSO emit a `calendar_view` with
  variant=compact and display_events=<denormalized snapshot>.
- When user asks "show me my schedule": emit `calendar_view` with
  variant=full and editable=true and events_path=<live>.

Material component (muno/src/components/widgets/CalendarView.tsx)
-----------------------------------------------------------------
- Imports Firebase JS SDK `onSnapshot` for live mode (verify the SDK
  is in package.json from Round 6b's MapView/listener wiring; add if
  missing).
- Variants:
  - compact: MUI List with date · time · type-icon · title · location.
  - full: MUI Table with sortable columns + edit pencils per row.
  - read_only: same as full minus the pencil column.
- Type → icon map: flight_depart/flight_arrive → FlightTakeoff/FlightLand;
  check_in/check_out → Hotel/MeetingRoom; activity → Event;
  user_note → Notes (all from @mui/icons-material).
- Edit pencil click: widget_cta_click event with {event_id, action: 'edit_event'}.
- date-fns for date formatting (already in deps).
- Test fixture exercises both static and live modes (mock onSnapshot
  if Firestore emulator isn't set up).
