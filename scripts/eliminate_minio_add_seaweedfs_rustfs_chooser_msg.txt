Architecture:
  Helm + bootstrap playbook accept `objectStore.kind: seaweedfs | rustfs`.
  Default: seaweedfs (mature, scale-proven).
  Both backends expose a Service named `object-store` on port 9000 in the
  configured namespace. Workers + server's NOETL_S3_ENDPOINT resolves to
  `http://object-store.<ns>.svc.cluster.local:9000` regardless of backend.
  Bucket convention preserved as `noetl`.

  Image tags:
    SeaweedFS: chrislusf/seaweedfs:3.x (latest stable)
    RustFS:    rustfs/rustfs:latest

  The noetl-side S3 client is backend-agnostic (boto3 with custom endpoint),
  so swapping backends is a deployment concern, not a code concern.
