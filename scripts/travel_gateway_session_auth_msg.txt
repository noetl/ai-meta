Context
-------
The original Round 9 (Cloudflare Access at the edge) was the WRONG
pattern. Cancelled before firing. The correct pattern is the one
already in production at repos/gui (admin UI behind mestumre.dev):

  Cloudflare Pages public SPA (travel.mestumre.dev)
          ↓
  Travel app login button
          ↓
  Auth0 SPA flow (existing PR #17 — preserved)
          ↓
  POST gateway.mestumre.dev/api/auth/login (Auth0 access token)
          ↓
  NoETL Auth0 login playbook issues session_token
          ↓
  Travel SPA stores session_token (same way GUI does)
          ↓
  All playbook execution calls carry session_token to gateway.

Single identity (kadyapam@gmail.com via Auth0) flows end-to-end.
Guest mode disabled in production builds; preserved for local dev via
Vite env flag.

Discovery is the most important phase
-------------------------------------
Codex inspects repos/gui FIRST, before writing any travel code. Map:
  - exact gateway login endpoint URL
  - HTTP method + request body shape (where the Auth0 token goes)
  - response shape (where the session_token lives, expires_at,
    user info)
  - storage mechanism (localStorage / sessionStorage / etc.)
  - HTTP header that carries the session_token to subsequent calls
  - 401 handling and logout flow

Land this map at `repos/travel/docs/auth/gateway-session-pattern.md`
BEFORE writing any travel code. The implementation that follows must
match what GUI actually does, byte-for-byte where reasonable.

Architectural rules
-------------------
  - Mirror GUI's pattern. Don't infer — inspect.
  - Auth0 SPA flow (PR #17) preserved and EXTENDED. The gateway
    exchange happens AFTER Auth0 login completes.
  - VITE_ALLOW_GUEST=true preserves guest mode for local dev.
    Unset (default) in production → no guest fallback.
  - No Cloudflare Access. No edge gating. The cancelled Round 9 stays
    cancelled. The cloudflare-access-token GCP secret can stay
    provisioned but isn't used by this round.
  - No Auth0 tenant changes. No DNS changes. No Cloudflare Pages
    settings changes (other than maybe ensuring VITE_ALLOW_GUEST
    isn't set in the prod env).
  - No new GCP secrets.
  - Read-only inspection of repos/gui. Do NOT modify gui.
  - Frontend Firestore live reads stay unauthenticated against v1
    permissive rules; tightening that is a SEPARATE follow-up round.
  - Never echo tokens (Auth0, gateway session_token, refresh) in
    stdout / logs / memory / result JSON. Length + 4-char prefix max.
  - If gateway exchange fails in Kadyapam's smoke: AMBER, leave PR
    open with the pattern doc + the actual gateway error captured.
  - PR via standard flow on noetl/travel main.
  - ai-meta pointer bumps local-only.
