  NoETL executes `kind: python` step code with separate globals and locals
  dicts. Helper functions defined at the top of a step's code body are
  visible to top-level statements but NOT to OTHER helper functions via
  closure, unless explicitly republished via `globals().update({...})`.

      ### Republish helpers through globals before calling them from other helpers

      [two-paragraph body: NoETL's separate globals/locals model;
       republish through globals().update({...}) after defining all helpers]

      Good:
      ```python
      def _helper_one():
          return _helper_two() + 1

      def _helper_two():
          return 42

      globals().update({
          '_helper_one': _helper_one,
          '_helper_two': _helper_two,
      })

      result = {'value': _helper_one()}
      ```

      Bad:
      ```python
      def _helper_one():
          # NameError at runtime: _helper_two is not visible inside _helper_one
          # because NoETL evaluates this snippet with separate globals and
          # locals.
          return _helper_two() + 1

      def _helper_two():
          return 42

      result = {'value': _helper_one()}
      ```
