Round goal
----------
travel.mestumre.dev is currently PUBLIC. The in-app Auth0 SPA flow
(noetl/travel#17) gates the UX but not the asset surface — anyone can
fetch the SPA bundle. Close the gap by configuring Cloudflare Access
at the edge of the Pages deployment, using the Auth0 IdP that's
already wired into Cloudflare Zero Trust for gateway.mestumre.dev.

All work via Cloudflare REST API from Codex. Token in GCP Secret
Manager (`cloudflare-access-token`), never echoed. Documentation goes
to repos/travel + repos/docs. No code changes.

Architectural rules
-------------------
  - Edge protection only. NO server-side JWT validation in this round.
  - In-app Auth0 SPA flow (PR #17) untouched. Defense-in-depth: edge
    gate + in-app token = two layers. UX consolidation is a separate
    follow-up.
  - Codex makes API calls via the CF token. Never echo token or
    session cookies in stdout / memory / result JSON.
  - Do NOT modify other Cloudflare Access apps. List + verify only.
  - Do NOT touch DNS or Cloudflare proxy settings on the travel record.
  - Do NOT call any Auth0 Management API. Kadyapam handled Auth0.
  - Rollback is `DELETE /accounts/{account_id}/access/apps/{app_uuid}`
    — Codex records the UUID prominently in the result JSON.
  - If site is still 200 (unprotected) after 60s post-policy retry:
    AMBER + STOP for diagnosis.
  - Documentation in repos/travel + repos/docs. No code in muno or ops.
  - PRs via standard flow; ai-meta pointer bumps local-only.
