Context:
  Final missing piece of the multi-provider arc. Today travel supports openai +
  anthropic (urllib direct) + vertex-ai (via mcp/vertex-ai playbook). Ollama was
  deferred because there was no NoETL playbook wrapping the in-cluster bridge —
  only a raw MCP server at `http://ollama-bridge.noetl.svc.cluster.local:8765/jsonrpc`.

  Phase 4 creates `automation/agents/mcp/ollama` as a NoETL playbook exposing the
  standard `chat_completion` MCP contract (identical surface to mcp/vertex-ai),
  then wires the travel runtime to invoke it via `tool: agent framework: noetl`.

  After this round, all four providers route through canonical interfaces:
    openai/anthropic → urllib direct in classify_via_http_provider (cloud APIs,
                       trivial auth, no benefit from playbook wrapping)
    vertex-ai        → mcp/vertex-ai playbook hop (complex GCP auth chain
                       belongs in playbook)
    ollama           → mcp/ollama playbook hop (in-cluster bridge wrapped in
                       canonical MCP playbook)

    metadata.name: chat_completion
    metadata.path: automation/agents/mcp/ollama
    agent: true, exposes_as_mcp: true, exposed_in_ui: false
    workload: model='gemma3:4b', messages=[], system='', temperature=0.1,
              ollama_endpoint='http://ollama-bridge.noetl.svc.cluster.local:8765/jsonrpc'
    workflow: one kind:python step that:
      - builds JSON-RPC envelope: {jsonrpc:'2.0', method:'tools/call',
        params:{name:'chat_completion', arguments:{model,messages,system,temperature}}, id:<int>}
      - POSTs via urllib to ollama_endpoint
      - parses MCP response, extracts text
      - returns chat_completion envelope shape MATCHING mcp/vertex-ai output:
        {status, content:[{type:'text',text}], isError, _meta:{backend:'ollama',model,usage,endpoint},
         usage, data:{_meta, ollama_response}, summary, text}
      - graceful 5xx via try/except urllib.error.HTTPError (same pattern as Amadeus MCP)
      - globals().update({...}) for helper visibility (NoETL Python idiom)
