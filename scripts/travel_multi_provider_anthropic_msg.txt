Authoring rules used (from the playbook authoring guide that just landed):

Design brief (full detail in task JSON `design_brief_for_codex`):

  1. KEYCHAIN ADDITION
     Add anthropic_token entry to travel runtime's keychain block, mirroring
     openai_token (kind: secret_manager, provider: gcp, scope: global,
     auth: "{{ gcp_auth }}", map.api_key: "{{ anthropic_secret_path }}").
     Bind unconditionally (no when:). New workload field anthropic_secret_path.

  2. MERGED CLASSIFY_INTENT STEP
     Replace classify_intent (kind: http) + parse_classification (kind: python)
     pair with a single classify_intent step (kind: python). The merged step:
       - Picks provider based on workload.ai_provider
       - Snaps to openai if anthropic chosen but token empty (graceful fallback)
       - Calls provider via urllib
       - Unwraps response (provider-specific shape: openai choices[0].message.content,
         anthropic content[0].text)
       - Parses JSON intent
       - Emits same fields parse_classification used to emit (intent, origin,
         destination, departureDate, adults, city, keyword, json_str) plus
         a NEW `effective_provider` field reflecting the actual provider
         that ran (may differ from workload.ai_provider when fallback fires)
       - On fallback: also sets `provider_fallback_reason` (e.g. 'anthropic
         token missing')

  3. DOWNSTREAM RENAMES
     log_classification SQL: parse_classification.X → classify_intent.X.
     log_classification arcs `when:` predicates: same rename.
     amadeus_call_flights / amadeus_call_locations / render_help /
     render_amadeus_failure inputs: same rename.
     render_* status pills now show effective_provider (the actual provider
     that ran) instead of workload.ai_provider.

  4. TUTORIAL 07 UPDATE
     repos/docs/docs/tutorials/07-travel-agent-with-widgets.md "Pluggable AI
     provider" section: replace OpenAI-only Jinja example with the merged
     python step's if/else pattern. Add try-it block:
       travel --provider anthropic flights from SFO to JFK on 2026-07-15
     Note Vertex AI / Ollama deferred (link to sync issue 2026-05-09).

Architectural rules (from the authoring guide):
  - Bare workload references in keychain (`{{ gcp_auth }}`, not `{{ workload.gcp_auth }}`)
  - No `when:` predicates on keychain entries
  - No Jinja conditionals on URL/headers/body — use python step for provider switch
  - Use urllib in kind:python for external calls (HTTP tool aborts on 5xx)
  - Render-emitting steps are workflow tails (no trailing audit chain)
  - Use `outcome` (not `status: failed`) for non-status semantic state
