  repos/ops/automation/agents/travel/runtime.yaml — five render_* steps:

    Each render_* step (render_flights, render_hotels, render_locations,
    render_activities, render_amadeus_failure) gets a best-effort audit

    INSERT shape:
      INSERT INTO travel_agent_events (execution_id, event_type, ai_provider, intent, payload)
      VALUES (%s, %s, %s, %s, %s::jsonb)

    Per-step values:
      execution_id: from input scope
      event_type:   'render_flights' / 'render_hotels' / 'render_locations' /
                    'render_activities' / 'render_amadeus_failure'

      intent:       classification.intent
      payload:      JSON dict with render_kind, render_type, envelope_status_code,
                    envelope_ok, envelope_total, provider_fallback_reason,
                    classified_intent (the full classify_intent dict)

    _cur, _audit_exc) to avoid clobbering the step's existing locals. If

    Phase 3 / hotels-activities rounds.
