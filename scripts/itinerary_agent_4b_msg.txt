Round goal
----------
Build the LLM-driven itinerary agent inside the muno repo at
`muno/playbooks/itinerary-planner.yaml`. Hybrid input: scripted widget
submissions AND free-form chat messages both feed the same per-turn
LLM extraction step. Tool dispatch through mcp/duffel + mcp/google-places
+ mcp/amadeus + mcp/firestore (event sourcing). Emits widget envelopes
matching the 23 schemas frozen in Round 4a/6a.

Pre-handoff: NONE — all dependencies already deployed and verified.

Runs PARALLEL with Round 6b (muno-material-widgets-6b). Disjoint paths
in muno — this round touches `muno/playbooks/` only; 6b touches
`muno/src/` only. Rebase before merging if the other PR lands first.

Per-turn loop (one event → zero-or-more emitted events)
------------------------------------------------------
1. Append the input event to chat_threads/{thread_id}/events.
2. LLM extraction step (JSON mode, temp=0):
     input: slot_state + last 10 events + new event
     output: { slot_updates, tool_requests, render_intent }
3. Apply slot_updates → write to chat_threads/{id}/slot_state.
4. For each tool_request: dispatch → record full request/response in
   events/api_calls/{call_id} (with header redaction).
5. LLM chat step (cheaper model):
     input: slot_state + tool_responses + render_intent
     output: { bot_message?, widgets: [<envelope>, ...] }
6. Validate each envelope against muno/playbooks/widget-contract/.
   Invalid → swap for a bot_text describing the agent's intent.
7. Append agent_widget_emit / agent_chat events.

Architecture decisions locked
-----------------------------
  - Hotels source: Amadeus (Duffel Stays sales-gated, Round 2).
  - flight_provider: duffel (default).
  - Production-mode endpoints FORBIDDEN. amadeus_env=test, duffel_env=test
    always. LLM cannot override.
  - All widget emissions validate against muno/playbooks/widget-contract/.
    Invalid → bot_text fallback. Never crash.
  - All events via mcp/firestore.append_event (mandatory header redaction).
  - Replay mode: temp=0 + model pinning + tool responses replayed from
    events/api_calls. LLM drift on chat text is tolerated; slot_updates
    and widget_type sequence must be byte-identical for clean replay.
